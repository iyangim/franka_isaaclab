# PPO 설정 파일 분석 및 작성 가이드 (skrl_ppo_cfg.yaml)

본 문서는 Franka Isaac Lab 프로젝트 내 3가지 강화학습 태스크(`reach`, `lift`, `stack`)의 PPO 에이전트 설정 파일(`skrl_ppo_cfg.yaml`) 구조를 분석하고, 향후 신규 태스크 생성 시 활용할 수 있는 작성 가이드를 제공합니다.

---

## 1. 태스크별 설정 비교 분석 (Comparison)

세 태스크는 작업 난이도에 따라 신경망의 크기(Layers), 학습률(Learning Rate), 학습 시간(Timesteps) 등의 파라미터가 다르게 튜닝되어 있습니다.

| 항목 | Reach (도달) | Lift (리프팅) | Stack (쌓기) |
| :--- | :--- | :--- | :--- |
| **은닉층 크기 (Layers)** | `[64, 64]` | `[256, 128, 64]` | `[256, 128, 64]` |
| **학습률 (Learning Rate)** | `1.0e-03` | `1.0e-04` | `1.0e-04` |
| **학습 에폭 수 (Learning Epochs)** | `5` | `8` | `8` |
| **총 훈련 타임스텝 (Timesteps)** | `24,000` | `36,000` | `36,000` |
| **엔트로피 가중치 계수** | `0.01` | `0.001` | `0.001` |
| **가치 손실 계수** | `1.0` | `2.0` | `2.0` |
| **보상 셰이핑 스케일** | `1.0` | `0.01` | `0.01` |
| **로그 저장 디렉토리** | `reach_franka` | `franka_lift` | `franka_lift` (※ 중복 설정 확인) |

### 💡 주요 분석 시사점
1. **신경망 복잡도 (Layers)**: Reach는 목표 좌표에 단순 도달하는 1단계 제어이므로 은닉층 노드가 `[64, 64]`로 컴팩트하게 구성된 반면, 물체를 잡아서 들어 올리고 쌓는 비선형성이 높은 Lift/Stack 제어에서는 더 풍부한 특징 추출을 위해 `[256, 128, 64]`의 깊고 넓은 구조를 취합니다.
2. **학습률 및 에폭**: 태스크가 어려울수록 세밀한 파라미터 업데이트가 필요하므로 학습률을 10배 작게 설정(`1.0e-04`)하고 학습 에폭 수를 늘려 학습 안정성을 도모합니다.
3. **보상 셰이핑 및 엔트로피**: 복잡한 태스크에서는 과도한 무작위 탐험을 억제하기 위해 엔트로피 가중치를 낮추고(`0.001`), 학습 초기 급격한 변동을 줄이기 위해 보상 스케일 값을 낮추어 튜닝했습니다.
4. **설정상의 버그 가능성**: Stack 태스크의 설정 파일에 저장 폴더명이 `directory: "franka_lift"`로 지정되어 있어, Stack 태스크의 학습 결과가 Lift 폴더에 덮어씌워질 우려가 있으므로 `franka_stack` 등으로 경로 분리가 권장됩니다.

## 2. YAML 설정 파일 블록 구성 및 작성 방법

skrl의 설정 파일은 에이전트 인스턴스화를 자동화하기 위해 크게 5가지 블록으로 구분하여 작성합니다.

```yaml
seed: 42  # 1. 시드 설정
models:   # 2. 신경망 모델 구조 설정
  ...
memory:   # 3. 재생 메모리(Buffer) 설정
  ...
agent:    # 4. PPO 알고리즘 하이퍼파라미터 설정
  ...
trainer:  # 5. 트레이너 실행 루프 설정
  ...
```

### 1) `models` 블록 작성 가이드
skrl의 `Model Instantiator` 인터페이스를 사용하여 텐서플로우/PyTorch 신경망을 설계합니다.
* **`separate`**: `True`로 설정하면 정책과 가치 네트워크가 가중치를 공유하지 않는 독립된 신경망으로 분리 생성됩니다.
* **`policy`**: 에이전트의 행동을 결정하는 확률적(Gaussian) 네트워크를 구성합니다.
  * **`input: OBSERVATIONS`**: 단일 에이전트 환경의 래퍼 규격과 연동되도록 반드시 `OBSERVATIONS`로 설정해야 합니다. (이전 `STATES` 설정 시 에러 발생)
  * **`layers`**: 뉴런 구조 리스트를 지정합니다 (예: `[256, 128, 64]`).
  * **`activations`**: `elu`, `relu`, `tanh` 등 활성화 함수를 정의합니다.
  * **`output`**: 최종 결정되는 행동 제어이므로 `ACTIONS`를 사용합니다.
* **`value`**: 상태 가치 평가 네트워크($V(s)$)를 정의하며 결정적(Deterministic) 구조를 사용합니다. 아웃풋은 항상 실수 스칼라 한 개이므로 `output: ONE`으로 작성합니다.

### 2) `memory` 블록 작성 가이드
환경 탐험 과정에서 수집된 에피소드 경험을 일시적으로 보관하는 버퍼를 정의합니다.
* 일반적으로 `class: RandomMemory`를 사용합니다.
* `memory_size`를 `-1`로 설정하면 아래 `agent: rollouts` 스텝 크기에 맞추어 메모리 용량이 자동 조정됩니다.

### 3) `agent` 블록 작성 가이드
PPO(Proximal Policy Optimization) 최적화에 필요한 수학적 파라미터들을 튜닝합니다.
* **`rollouts`**: 한 번의 학습 업데이트 전에 환경으로부터 데이터를 수집할 단계(Step) 수입니다.
* **`discount_factor` ($\gamma$)**: 미래의 보상을 현재 가치로 환산할 할인율입니다. (일반적으로 `0.99`)
* **`lambda`**: GAE(Generalized Advantage Estimation)의 편향-분산 트레이드오프 조절 계수입니다. (일반적으로 `0.95`)
* **`learning_rate_scheduler`**: `KLAdaptiveLR`을 사용하면 훈련 도중 타깃 KL 임계값(`kl_threshold: 0.01`)을 벗어날 때 자동으로 학습률의 보폭을 조절합니다.
* **`state_preprocessor`**: 에이전트 입력 셰이프를 정규화하여 학습 수렴성을 높이기 위해 `RunningStandardScaler`를 사용합니다.

### 4) `trainer` 블록 작성 가이드
학습을 실행할 트레이너 클래스와 전체 학습 규모를 결정합니다.
* **`class: SequentialTrainer`**: 순차 연산 기반의 트레이너를 실행합니다.
* **`timesteps`**: 학습을 총 진행할 최종 스텝 수를 지정합니다.
