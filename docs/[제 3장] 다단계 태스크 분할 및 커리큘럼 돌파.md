# [제 3장] 다단계 태스크 분할 및 커리큘럼 돌파
## (Phase 3 & 4: Multi-Stage & Curriculum)

본 장은 로봇 강화학습에서 수렴 난이도가 가장 높은 Long-Horizon(장기 의존성) 및 희소 보상(Sparse Reward) 기반의 조작 과제를 해결하기 위한 시스템 설계도입니다. 물체를 집어서 다른 물품 위에 쌓는 **Pick-and-Stack** 과제(예: `Template-Stack-v0`)는 무작위 탐색 공간이 매우 넓어 단일 보상 함수만으로는 수렴 확률이 0%에 수렴합니다. 

이를 극복하기 위해 물리적 상태에 따라 보상 체계를 조건부로 전환하는 **PyTorch 텐서 기반 게이팅 메커니즘**과 하위 지능을 상속하는 **커리큘럼 웜스타트(Warm-start)** 전략을 실전 소스코드 레벨에서 상세히 규정합니다.

## **3.1 Long-Horizon 조작 과제의 수렴 병목 및 상태 분할(Task Matrix)**

장기 의존성 과제의 핵심 병목은 '희소 보상' 문제입니다. 에이전트가 우연히 물체를 잡고 들어 올려 매대에 쌓을 확률은 극도로 낮으므로, 전체 공정을 논리적 인과관계에 따라 5개 스테이지로 분할하고 각 단계의 활성 마스크를 정의하는 매트릭스를 시스템 아키텍처로 선언합니다.

| 스테이지 | 하위 목표 (Sub-goal) | 진입 트리거 조건 (Trigger Condition) | 핵심 활성화 보상 (Active Rewards) |
| :--- | :--- | :--- | :--- |
| **Stage 1** | **Reach**: 큐브 1 접근 | 에피소드 시작 직후 초기 상태 | 말단 작동기(EE)와 큐브 1 사이의 자석 커널 거리 보상 |
| **Stage 2** | **Grasp & Lift**: 파지 및 고도화 | EE와 큐브 1 밀착 완료 ($d_{ee\_c1} < 3\text{cm}$) | 그리퍼 압착 명령 유도, 큐브 1 Z축 고도 상승 보상 및 파지 보너스 |
| **Stage 3** | **Move**: 큐브 2 영역 이동 | 큐브 1 지면 이탈 완료 ($z_{cube1} - table\_z > 12\text{cm}$) | 큐브 1과 큐브 2 간의 수평 ($X, Y$) 거리 도달 보상 |
| **Stage 4** | **Align & Stack**: 수직 적재 | 큐브 1-2 수평 오차 진입 ($d_{c1\_c2, xy} < 3.5\text{cm}$) | 두 오브젝트의 $X, Y$ 중심축 일치 정렬 보상 및 하강 유도 |
| **Stage 5** | **Release & Verify**: 안정성 검증 | 큐브 1-2 전체 3D 거리 안착 ($d_{c1\_c2} < 5\text{cm}$) | 그리퍼 손가락 개방 유도, 로봇 팔 탈착 후퇴 및 안착 유지 보너스 |

## **3.2 5대 스테이지별 구체적 물리 적용 및 실패 방어 전략**

### **1단계: Stage 1 - Reach Mode (공간 탐색 및 목적물 접근)**
- **목표 및 물리적 의미**: 로봇의 말단 작동기(EE)가 공간 내 무작위로 배치된 첫 번째 진열 물품(Cube 1)의 상단 영역까지 그리퍼를 정렬하며 접근하는 단계입니다.
- **수학적 트리거 조건**: 에피소드 초기화 직후에는 무조건 True 마스크로 시작합니다.
  - 타깃 도달 판정선: $d_{ee\_c1} = \text{torch.norm}(c1\_pos - ee\_pos, dim=-1) \ge 0.03\text{m}$
- **실전 보상 성형 매핑**: 오직 Cube 1 접근 보상 하나만 활성화하고, 파지 보너스나 이동 보상은 $0$으로 완전히 잠금(Gating)합니다.
  - $R_{stage1} = w_{r} \cdot \exp(-d_{ee\_c1}^2 / \sigma_{reach}^2)$ (추천 가중치 $w_r = 1.0$, 초기 $\sigma_{reach} = 0.1$)
- **실패 방어 전략**: 이 단계에서는 절대 그리퍼 닫기 명령(Action)에 패널티를 주거나 보상을 주어서는 안 됩니다. 팔을 뻗기도 전에 그리퍼를 닫고 굳혀버리는 **Frozen Policy** 현상이 발생하기 때문입니다.

### **2단계: Stage 2 - Grasp & Lift Mode (정밀 파지 및 수직 고도 확보)**
- **목표 및 물리적 의미**: 물체에 EE가 밀착했을 때 강체 간의 힘 폐쇄(Force Closure)를 형성하여 물품을 움켜쥐고, 테이블 마찰력을 이겨내며 Z축 방향으로 들어 올리는 단계입니다.
- **수학적 트리거 조건**:
  - 조건 A (도달 완료): $d_{ee\_c1} < 0.03\text{m}$
  - 조건 B (그리퍼 압착): Franka 양 손가락 관절 포지션 합산 데이터인 $gripper\_width$가 물체 두께 범위 내에 상주 ($0.03\text{m} < gripper\_width < 0.075\text{m}$)
  - 플래그 연산: $is\_c1\_grasped = 조건 A \;\&\; 조건 B$
- **실전 보상 성형 매핑**: $is\_c1\_grasped$가 True가 되는 순간 파지 성공 보상($+2.0$)을 더해주고, Z축 고도 상승 보상을 개방합니다.
  - $R_{stage2} = R_{grasp\_bonus} + w_{l} \cdot (c1\_pos[:, 2] - table\_z)$ (추천 가중치 $w_l = 4.0$)
- **실패 방어 전략**: 초기 수렴 단계에서 물체가 조금만 미끄러져도 조건이 깨지므로, 초기 학습 시에는 그리퍼 폭 허용 범위를 일부러 너그럽게 잡고, 물리 에셋(`RigidObjectCfg`) 내 정적 마찰 계수를 1.0 이상으로 설정하여 시뮬레이터 내 물체 탈출을 방어합니다.

### **3단계: Stage 3 - Move Mode (수평 공간 이송 및 간섭 회피)**
- **목표 및 물리적 의미**: 들어 올린 물품을 매대의 목표 진열 Spot(Cube 2의 상단 영역)까지 테이블 경계면과의 충돌 없이 안전하게 수평 이송하는 단계입니다.
- **수학적 트리거 조건**: $is\_c1\_grasped == True$를 유지하며, 동시에 큐브의 고도가 상승했을 때 활성화됩니다.
  - 리프트 트리거 선: $z_{cube1} - table\_z > 0.12\text{m}$
- **실전 보상 성형 매핑**: Stage 1의 접근 보상을 완전히 차단하고, Cube 1과 Cube 2 간의 거리를 좁히는 커널 보상을 활성화합니다.
  - $R_{stage3} = w_{m} \cdot \exp(-d_{c1\_c2}^2 / \sigma_{move}^2)$ (추천 가중치 $w_m = 1.0$, 초기 $\sigma_{move} = 0.15$)
- **실패 방어 전략**: 리프트 고도 판정 임계값이 지나치게 높으면(예: $0.2\text{m}$), 로봇이 물건을 조금만 들고 옆으로 이동하려 할 때 보상이 차단되어 굳어버립니다. 정체 현상 발생 시 트리거 높이를 $0.08 \sim 0.10\text{m}$ 수준으로 완화해야 수평 이송 제어가 수렴합니다.

### **4단계: Stage 4 - Align & Stack Mode (중심축 일치 및 데드존 안착)**
- **목표 및 물리적 의미**: 목표 진열 Spot 상공에 도달했을 때 물품이 매대 밖으로 이탈하거나 쓰러지지 않도록 수직 동심축을 일치시키고 하강시키는 정밀 조작 단계입니다.
- **수학적 트리거 조건**:
  - 수평 정렬 완료 판정선: $d_{c1\_c2, xy} < 0.035\text{m}$
- **실전 보상 성형 매핑**: 두 물체의 중심축이 어긋날 때 감점을 부여하는 동심축 일치 보상($r_{align}$)하고 안정적인 하강 보상($r_{descend}$)을 부여합니다.
  - $R_{stage4} = 4.0 + r_{align} + r_{descend}$
  - $r_{align} = \exp(-d_{c1\_c2, xy}^2 / \sigma_{align}^2)$ (초기 $\sigma_{align} = 0.04$)
  - $r_{descend} = \text{torch.where}(is_aligned, 5.0 - (c1_pos[:, 2] - c2_pos[:, 2]) * 10.0, torch.zeros_like(total_reward))
- **실패 방어 전략**: 이 구역에서 로봇이 물체를 던지거나 세게 밀치며 충돌을 발생시키는 경우가 많습니다. 액션 변화량을 감점하는 `action_rate_l2` 패널티의 가중치를 정밀 제어 시점에 맞춰 보정하여 정렬 시의 과속 거동을 감속시켜야 합니다.

### **5단계: Stage 5 - Release & Verify Mode (힘 완화 및 구조적 안정성 평가)**
- **목표 및 물리적 의미**: 적재를 완료한 후 그리퍼를 열어 물품과의 접촉을 해제하고, 로봇 팔이 후퇴하더라도 진열된 물품이 무너지지 않고 버티는지 자립 안정성을 평가하는 단계입니다.
- **수학적 트리거 조건**:
  - 완전 적재 성공 판정선: $d_{c1\_c2} < 0.05\text{m}$
  - 물체 적재 완료(Stacked) 상태: $is\_stacked = (c1\_pos[:, 2] - c2\_pos[:, 2]) > 0.045$
- **실전 보상 성형 매핑**: 적재를 완료하고 그리퍼가 물건을 안정적으로 놓아준 상태에서 최종 도달 보상과 안착 유지 보너스를 극대화합니다.
  - $R_{stage5} = \text{torch.where}(is\_stacked \; \& \; (d_{c1\_c2, xy} < 0.025), 15.0, 0.0)$
- **실패 방어 전략**: 만약 손을 떼는 순간 적재물이 미끄러져 무너지면 에피소드를 즉시 강제 종료(`terminations`)하여 정책 신경망에 오동작 가중치가 전파되는 것을 원천 차단해야 합니다.

## **3.3 PyTorch 텐서 기반 조건부 보상 잠금(Gating Mask) 구현 명세**

Isaac Lab 런타임상에서는 수백~수천 개의 가상 환경이 GPU 대규모 병렬 연산으로 구동되므로, 조건 분기에 파이썬의 `if-else` 구문을 사용하면 연산 대역폭이 완전히 붕괴됩니다. 모든 분기 조건은 PyTorch 텐서 연산인 `torch.where` 형식으로 강제 구현되어야 병렬성이 유지됩니다.

다음은 `mdp/rewards.py` 파일에 기동되는 실전 아키텍처 스크립트입니다.

```python
import torch
from omni.isaac.lab.assets import Articulation, RigidObject
from omni.isaac.lab.envs import ManagerBasedRLEnv

def check_grasp_success(env: ManagerBasedRLEnv, robot_name: str = "robot") -> torch.Tensor:
    """로봇 그리퍼의 손가락 조인트 센서 데이터를 역추적하여 파지 유무를 판정하는 함수
      
    차원 명세:
        - joint_pos: [num_envs, num_joints] -> 프란카의 경우 7, 8번 인덱스가 핑거 관절
        - 리턴 텐서: [num_envs] 형태의 torch.bool
    """
    robot: Articulation = env.scene[robot_name]
    
    # 1. 프란카 판다 로봇의 finger_joint1, finger_joint2 현재 포지션 추출
    finger_pos = robot.data.joint_pos[:, 7:9]
    gripper_width = finger_pos[:, 0] + finger_pos[:, 1]
    
    # 2. 파지 판정 임계값 필터링 (큐브 두께인 5cm 범주를 안정적으로 쥐고 있는지 검증)
    # 완전 닫힘(0.0)이나 완전 개방(0.08 이상)이 아닐 때 True를 반환하여 정밀 제어 유도
    grasped_mask = (gripper_width > 0.03) & (gripper_width < 0.075)
    
    return grasped_mask

def multi_stage_stacking_reward(
    env: ManagerBasedRLEnv, 
    robot_name: str = "robot", 
    cube1_name: str = "cube1", 
    cube2_name: str = "cube2",
    table_name: str = "table"
) -> torch.Tensor:
    """5단계 상태 전이 트리거 플래그 기반 논리 통합 보상 함수
    
    Returns:
        torch.Tensor: [num_envs] 크기의 배치 단위 실시간 동기화 보상 텐서
    """
    # [시스템 자산 인스턴스 매핑]
    robot: Articulation = env.scene[robot_name]
    cube1: RigidObject = env.scene[cube1_name]
    cube2: RigidObject = env.scene[cube2_name]
    table: RigidObject = env.scene[table_name]
    
    # [기하학적 3차원 위치 벡터 수집 - Device: cuda:0]
    ee_pos = robot.data.ee_w_pos          # 말단 작동기 중심 좌표 [num_envs, 3]
    c1_pos = cube1.data.root_pos_w        # 적재 대상 물체 1 중심 좌표 [num_envs, 3]
    c2_pos = cube2.data.root_pos_w        # 기반 물체 2 중심 좌표 [num_envs, 3]
    table_pos = table.data.root_pos_w      # 테이블 중심 좌표 [num_envs, 3]
    
    # [유클리드 거리 및 상대 거리 배치 연산]
    dist_ee_to_c1 = torch.norm(c1_pos - ee_pos, dim=-1)      # EE와 큐브1 간 거리 [num_envs]
    dist_c1_to_c2 = torch.norm(c2_pos - c1_pos, dim=-1)      # 큐브1과 큐브2 간 전체 거리 [num_envs]
    dist_c1_to_c2_xy = torch.norm(c2_pos[:, :2] - c1_pos[:, :2], dim=-1)  # 수평 오차 거리 [num_envs]
    
    # [각 스테이지별 진입 제어 Boolean 마스크 연산]
    is_c1_grasped = check_grasp_success(env, robot_name)
    is_aligned = dist_c1_to_c2_xy < 0.035                    # 수평 동심축 3.5cm 이내 정렬 여부
    is_stacked = (c1_pos[:, 2] - c2_pos[:, 2]) > 0.045       # 수직 적재 안착 및 고도 차이 여부
    
    # 종합 보상 저장용 마스터 버퍼 초기화
    total_reward = torch.zeros(env.num_envs, device=env.device, dtype=torch.float32)
    
    # ------------------------------------------------------------------
    # Stage 1: Reach Mode (물체를 잡지 못했을 때 - 자석 커널 보상)
    # ------------------------------------------------------------------
    r_reach = torch.exp(-torch.square(dist_ee_to_c1) / (0.1 ** 2))
    total_reward = torch.where(~is_c1_grasped, r_reach, total_reward)
    
    # ------------------------------------------------------------------
    # Stage 2 & 3: Grasp & Move Mode (파지 성공 후 수평 이송 가속화)
    # ------------------------------------------------------------------
    r_lift = 2.0 + (c1_pos[:, 2] * 4.0)  # 리프팅 유도 가중치
    r_move = torch.exp(-torch.square(dist_c1_to_c2) / (0.15 ** 2))
    
    move_condition = is_c1_grasped & ~is_aligned
    total_reward = torch.where(move_condition, r_lift + r_move, total_reward)
    
    # ------------------------------------------------------------------
    # Stage 4: Align & Stack Mode (목표 상공 진입 시 동심축 일치 유도)
    # ------------------------------------------------------------------
    r_align = torch.exp(-torch.square(dist_c1_to_c2_xy) / (0.04 ** 2))
    r_descend = torch.where(is_aligned, 5.0 - (c1_pos[:, 2] - c2_pos[:, 2]) * 10.0, torch.zeros_like(total_reward))
    
    stack_condition = is_c1_grasped & is_aligned & ~is_stacked
    total_reward = torch.where(stack_condition, 4.0 + r_align + r_descend, total_reward)
    
    # ------------------------------------------------------------------
    # Stage 5: Release & Verify Mode (적재 완료 후 안정성 평가 및 해제 보너스)
    # ------------------------------------------------------------------
    r_verify = torch.where(is_stacked & (dist_c1_to_c2_xy < 0.025), torch.ones_like(total_reward) * 15.0, torch.zeros_like(total_reward))
    total_reward = torch.where(is_c1_grasped & is_stacked, r_verify, total_reward)
    
    # 실시간 디버깅용 로그 전수 프린트 (배치 내 0번 가상환경 기준 모니터링 덤프)
    if env.common_step_counter % 100 == 0:
        is_c1_lifted = (c1_pos[:, 2] - table_pos[:, 2]) > 0.12
        print(f"[STAGE DUMP] Step: {env.common_step_counter} | "
              f"Grasped: {is_c1_grasped[0].item()} | Lifted: {is_c1_lifted[0].item()} | "
              f"Aligned: {is_aligned[0].item()} | Stacked: {is_stacked[0].item()} | "
              f"Reward: {total_reward[0].item():.4f}")
              
    return total_reward
```

### **태스크 환경 설정 통합 연동 (stack_env_cfg.py)**

## **3.4 커리큘럼 웜스타트(Warm-start) 가동 및 정책 네트워크 상속 전술**

고난도 Stack 과제를 베이스라인 가중치 없이 무작위 탐색(Cold-start)으로 훈련시키면 정책 최적화가 쉽게 발산합니다. 하위 태스크의 성과 가중치 자산을 이어받는 시스템 운영 절차입니다.

1. **단계 전이 기준 정의 (Gating Threshold)**: 
   `Template-Reach-v0` 태스크의 최근 100 에피소드 이동 평균 성공률(Success Rate) 지표가 **70% 이상**을 안정적으로 돌파하는 순간의 최상위 가중치 파일(`best_agent.pt`)을 복사하여 독립 아카이빙합니다.

2. **정책 신경망 상속 및 로드 명령어**: 
   상위 난이도 태스크인 `Template-Stack-v0`를 가동할 때, `--resume` 인자와 함께 직전 마스터 가중치 파라미터 경로를 주입하여 가중치를 상속받습니다.
   
   ```bash
   conda activate isaaclab
   cd ~/franka_isaaclab
   
   # python.sh 인터프리터를 통한 웜스타트 학습 실행
   ~/smart-shelf-robot/third_party/IsaacLab/_isaac_sim/python.sh scripts/skrl/train.py \
     --task=Template-Stack-v0 \
     --resume \
     --checkpoint=logs/skrl/reach_franka/best_agent.pt \
     --num_envs=64 \
     --headless
   ```

3. **시각/상태 인코더 프리징 유도**: 
   상속 구동 시 SKRL 트레이너 내 가중치 마샬링 버퍼는 공간 기하학 감각을 습득한 앞단의 상태 특징 추출 레이어(Feature Extractor)의 가중치를 고정 상태로 보존하고, 하단 모터 제어 액터 헤드(Actor Head) 영역 위주로 정밀 미세 튜닝을 실행하므로 수렴 속도가 수십 배 단축됩니다.

## **3.5 텐서보드(TensorBoard) 분석 및 실시간 트러블슈팅 매트릭스**

5단계 분할 공정이 시뮬레이터 내부에서 논리적으로 맞아떨어지며 수렴하고 있는지는 일반적인 Total Reward 곡선만 보면 판정하기 어렵습니다. 학습 실행 후 텐서보드 상에서 개별 보상 컴포넌트 차트가 시간 축에 따라 순차적으로 깨어나는 **계단식 도약 패턴(Stepwise Awakening)**이 관찰되어야 분할 엔지니어링이 정상 동작하고 있는 것입니다.

```
보상 점수 (Reward)
   ^
   │                                           ┌─── Stage 5 (Release)
   │                                   ┌───────┘ 
   │                           ┌───────┘ Stage 4 (Align)
   │                   ┌───────┘ Stage 3 (Move)
   │           ┌───────┘ Stage 2 (Grasp)
   │   ┌───────┘ Stage 1 (Reach)
   └───┴───────────────────────────────────────────────────────> 학습 타임라인 (Steps)
      0k      10k     20k     30k     40k
```

### **콘솔 크래시/로그 증상별 즉각 처방 매트릭스**

| 콘솔 크래시 / 로그 증상 | 원인 진단 (Root Cause) | 코드 레벨 즉각 디버깅 처방 |
| :--- | :--- | :--- |
| **RuntimeError: Expected all tensors to be on the same device** | 연산 변수 간의 디바이스 메모리 상충 (CPU vs GPU 주소 혼선) | 새로 선언한 `torch.zeros` 또는 상수 텐서 정의 시 반드시 `device=env.device` 파라미터를 강제 매핑하여 자원을 일치시킵니다. |
| **로그 확인 시 Aligned 플래그가 영원히 False 유지** | 수평 정렬 기하학 축 판단 시 Z축 높이 간섭 배제 누락 오류 | 수평 오차 연산부 텐서 조작 시 반드시 슬라이싱 기법(`c2_pos[:, :2] - c1_pos[:, :2]`)을 사용하여 고도 성분을 완벽히 격리해야 축 정렬 지능이 활성화됩니다. |
| **ValueError: Shape mismatch inside memory batcher buffer** | 보상 함수 최종 출력값의 배치 차원 붕괴 현상 (차원 축소 누락) | 유클리드 거리를 산출하는 `torch.norm` 또는 `torch.sum` 연산자 인자에 무조건 `dim=-1`을 할당하여 최종 출력 차원 구조를 `[num_envs]` 형태의 1차원 벡터로 강제 조율합니다. |

### **다단계 조작 정체 구간별 해결 매트릭스**

| 터미널 플래그 판정로그 상태 | 병목 원인 분석 (Root Cause) | 즉각 엔지니어링 수정 조치 (Immediate Action) |
| :--- | :--- | :--- |
| **Reach: True <br>Grasped: False <br>Lifted: False** | 그리퍼 압착 범위 조건 협소화 및 모터 파지력(마찰계수) 부족 오류 | `check_grasp_success` 내의 $finger\_width$ 범위를 $0.03 \sim 0.08$로 확장하고, 에셋 설정 파일에서 큐브의 $static\_friction$ 계수를 1.0 이상으로 상향 조율합니다. |
| **Reach: True <br>Grasped: True <br>Lifted: False** | Z축 리프팅 고도 보상 임계값 조건이 실제 도달력 대비 너무 엄격함 | Stage 3 진입 마스크 조건의 Lift Trigger 임계값 수치를 기존 0.15에서 $0.08 \sim 0.10$ 수준으로 완화 조정하여 다음 스테이지 보상을 강제 개방합니다. |
| **Grasped: True <br>Lifted: True <br>Move/Align: 계속 0 정체** | 보상 게이팅 텐서 분할 결합부의 마스킹 수식 로직 크래시 | `torch.where`에 결합된 Boolean 조건 변수 간의 비트 연산자(`&`, `~`, `|`) 적용 우선순위 괄호 처리가 올바른지 확인합니다. |
